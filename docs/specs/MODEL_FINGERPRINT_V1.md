# Model Fingerprint Artifact — Specification v1

## Overview

A **model fingerprint** is a vault artifact (`kind: "profile"`) that captures
the complete behavioral identity of a model×quantization combination as
measured by the LemWorld behavioral research platform. It serves as the
canonical data contract between three systems:

- **LemWorld** (producer): behavioral sweeps generate fingerprints
- **LensForge** (analyst): fingerprints are queried for research analysis
- **ThreadForge** (consumer): fingerprints drive production routing decisions

## Payload Schema

```
schema:               "model_fingerprint.v1"  (literal, required)

model_family:         string    e.g. "gemma", "llama", "falcon3", "qwen"
model_name:           string    e.g. "gemma-2-27b-it"
model_size_b:         number    parameter count in billions
architecture:         string?   "transformer" | "ssm" | "hybrid" (optional)

quant_method:         enum      "fp16"|"bf16"|"fp32"|"gptq"|"awq"|"gguf"|"exl2"|"none"
quant_level:          enum      "fp16"|"bf16"|"fp32"|"int8"|"int4"|"int3"|"int2"|
                                "q8_0"|"q6_k"|"q5_k_m"|"q4_k_m"|"q3_k_m"|"q2_k"|"none"

behavioral_vector:    object    { contain, attract, release, repel, transform }
                                each: number [0, 1], sum ≈ 1.0
behavioral_profile:   enum      "guardian"|"diplomat"|"negotiator"|"liberator"|
                                "healer"|"sentinel"|"catalyst"|"altruist"|"balanced"

quant_verdict:        enum      "safe"|"degraded"|"broken"|"untested"
quant_delta:          number    BV euclidean distance from FP16 baseline [0, 2]
baseline_card_hash:   string?   hash of the FP16 baseline card (optional)

ap_metrics:           object    { vocabulary_richness, creative_output, snapback_score }
                                each: number [0, 1]

throughput:           object    { tok_per_sec, gpu_tested, vram_consumed_gb,
                                  max_context_tested, tensor_parallel?, speculative_draft? }

routing_fitness:      string[]  task categories this model is fit for
routing_exclusions:   string[]  task categories to avoid

sweep_source:         object    { lem_issue, sweep_date (YYYY-MM-DD),
                                  scenarios_tested, temperatures_tested? }
```

## Identity Rule

Follows the vault's universal identity rule:

```
id = sha256(canonicalize({
  version: "artifact_v1",
  kind: "profile",
  payload: <full fingerprint payload>,
  tags: <sorted tag array>,
  refs: <ref array>
}))
```

The payload is fully deterministic: same model tested with the same results
on the same scenarios with the same sweep date produces the same hash.
Environmental fields (hostname, GPU serial, instance ID, wall-clock timestamp)
belong in the envelope's `source` field, which is excluded from the hash.

## sweep_source and the Hash

`sweep_source.sweep_date` is a measurement date (ISO date, not datetime).
It is included in the hash because it identifies the measurement event: the
same model might be re-tested on a different date with updated scenarios,
producing a legitimately different card. This is not temporal contamination —
it is measurement identity.

Fields that would be temporal contamination (and are therefore banned):
`created_at`, `updated_at`, `timestamp`, `now`, etc.

## Tag Contract

Tags are generated by `buildFingerprintTags()` and enable efficient queries:

```
schema:model_fingerprint.v1      — all fingerprints
model:gemma-27b                  — model family + size
family:gemma                     — model family only
quant:int4                       — quantization level
method:awq                       — quantization method
verdict:degraded                 — quant verdict
profile:healer                   — behavioral profile
arch:transformer                 — architecture (if provided)
fit:<category>                   — routing fitness categories
tier:bronze-eligible             — ≥100 tok/s and not broken
tier:silver-eligible             — ≥40 tok/s and not broken
tier:gold-eligible               — not broken
```

Tags are sorted for canonical stability (identical tag content → identical hash).

## Refs

- If `baseline_card_hash` is set, the card carries a ref to its FP16 baseline:
  `{ kind: "profile", id: "<baseline_hash>" }`
- Otherwise refs is empty.

This creates a graph: quant variant → FP16 baseline. You can walk refs to
find the baseline for any quantized card.

## ThreadForge Tier Mapping

The vault's existing tier system maps directly to ThreadForge NFTs:

| Vault Tier | ThreadForge Tier | Artifact Cap | Kind Restrictions |
|------------|-----------------|--------------|-------------------|
| Bronze     | Bronze NFT      | 1,000        | fact, event, tool_obs only |
| Silver     | Silver NFT      | 10,000       | All kinds + promotion + blessing |
| Gold       | Gold NFT        | Unlimited    | Full access |

Bronze NFT holders can read fingerprint cards (via `vault.search` + `vault.get`)
but cannot write them (profile kind is restricted at Bronze tier). This is
correct: only the research pipeline writes fingerprints; consumers read them.

## Behavioral Vector Interpretation

| Verb      | Meaning in RVG Context                     | ThreadForge Routing Signal         |
|-----------|--------------------------------------------|------------------------------------|
| Contain   | Protect, guard, preserve, stabilize        | Security, moderation, compliance   |
| Attract   | Persuade, negotiate, build coalitions      | Customer engagement, sales, chat   |
| Release   | Share, give, liberate, distribute          | Creative writing, content gen      |
| Repel     | Reject, defend boundaries, push back       | Adversarial tasks, red-teaming     |
| Transform | Change, adapt, restructure, innovate       | Code refactoring, format conversion|

## Quant Verdict Thresholds

| Verdict  | BV Delta    | Description                              |
|----------|-------------|------------------------------------------|
| safe     | < 0.05      | Behavioral identity preserved            |
| degraded | 0.05 – 0.20 | Measurable shift, still usable          |
| broken   | > 0.20      | Profile flip or severe behavioral change |
| untested | N/A         | FP16 baseline only, no quant comparison  |

Critical finding: VR (vocabulary richness) and BV (behavioral vector) diverge
under quantization. A model can be VR-SAFE but BV-BROKEN (e.g. Llama 70B).
The fingerprint carries both metrics; the quant_verdict is always BV-based.

## Example Vault Operations

### Store a fingerprint

```typescript
import { buildFingerprintPutInput } from "./vault/fingerprint_schema.js";
import { vaultPut } from "./vault/store.js";

const result = await vaultPut(
  buildFingerprintPutInput(fingerprintData, {
    agent: "lemworld-sweep",
    tool: "rv_battery",
    run_id: "run_2026_03_15_001",
  })
);
// result.id = content-addressed hash
// result.created = true (first time) or false (dedup)
```

### Query by model + quant

```typescript
const results = await vaultSearch({
  tags: ["family:gemma", "quant:int4"],
  search_mode: "lexical",
});
```

### Find all broken models

```typescript
const broken = await vaultSearch({
  tags: ["verdict:broken"],
  search_mode: "lexical",
  limit: 100,
});
```

### Find production-eligible models for a tier

```typescript
const goldModels = await vaultSearch({
  tags: ["tier:gold-eligible"],
  search_mode: "lexical",
  limit: 50,
});
```

### Verify a card hash (Carapace pattern)

```typescript
const card = await vaultGet(cardHash);
if (card) {
  // Hash verified — this card exists and is authentic
  const fp = card.payload as ModelFingerprintPayload;
  console.log(`Model: ${fp.model_name}, Profile: ${fp.behavioral_profile}`);
}
```

## Test Requirements

All fingerprint artifacts must pass:

1. Deterministic double-run hash test
2. Prohibited key rejection (temporal/env)
3. Proto pollution rejection
4. Tag generation correctness
5. Vault round-trip (put → get → same hash)
6. Deduplication (same payload → same artifact ID)
7. Baseline ref wiring

See `tests/fingerprint.test.ts`.
